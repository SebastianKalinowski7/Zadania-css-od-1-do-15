1) Gdzie wstawiany jest styl zewnętrzny, jaki znacznik jest za to odpowiedzialny oraz ile razy może on występować w dokumencie jako podłączenie stylu CSS? Styl zewnętrzny umieszczany jest w nagłówku dokumentu HTML, między znacznikami a . Za jego podłączenie odpowiada znacznik: html Kopiuj Edytuj Znacznik może występować dowolną ilość razy, dla każdego stylu osobno. 2) Składnia teoretyczna html Kopiuj Edytuj 3) Jaka jest najprostsza forma wstawiania stylu zewnętrznego? Najprostsza forma to pojedyncza linijka wstawiona w sekcji dokumentu: html Kopiuj Edytuj 4) Ile arkuszy styli może być dołączonych do jednego dokumentu HTML oraz jaka zasada musi być zachowana przy większej ilości dołączanych styli? Można dołączyć dowolną liczbę arkuszy stylów. Każdy z nich powinien mieć osobny znacznik . Zasada: Jeśli kilka arkuszy zawiera style dla tych samych elementów, ważniejsze będą style z arkusza dołączonego później. 5) Co się dzieje w przypadku konfliktów? W przypadku konfliktu (czyli gdy te same selektory i cechy występują w różnych arkuszach), priorytet mają style z arkusza dodanego później. Przykład: html Kopiuj Edytuj 6) Jak są tworzone i edytowane pliki arkuszy z użyciem edytora tekstowego? Plik CSS to zwykły plik tekstowy z rozszerzeniem .css. Można go tworzyć i edytować w edytorach takich jak Notatnik, Notepad++, VS Code, Sublime Text itd. Przykład: css Kopiuj Edytuj body { background-color: lightgray; font-family: Arial; } 7) Jak są tworzone i edytowane pliki arkuszy z użyciem specjalnego edytora CSS? Do edytowania stylów można też używać specjalnych edytorów CSS, np.: Visual Studio Code (z dodatkami do CSS), Brackets, WebStorm, Stylizer. Takie narzędzia ułatwiają podpowiedzi składni, kolorowanie kodu i szybki podgląd zmian. 8) Jakie rozszerzenie posiada plik będący zewnętrznym arkuszem stylów? Plik zewnętrznego arkusza stylów posiada rozszerzenie: Kopiuj Edytuj .css Przykład: style.css 9) Jakie są zalety zewnętrznego arkusza stylów? Spójność wyglądu stron – wszystkie strony mogą mieć wspólny styl. Łatwość aktualizacji – zmieniając jeden plik .css, aktualizujemy wygląd wszystkich stron. 10) Uwagi na temat pisania w CSS Każda cecha i wartość muszą być zakończone średnikiem. Selektory muszą być poprawnie zapisane, bez literówek. Plik .css powinien być czysty i czytelny, najlepiej dobrze sformatowany. Warto grupować style dla przejrzystości: css Kopiuj Edytuj h1, h2, h3 { font-family: Arial; color: #333; }
